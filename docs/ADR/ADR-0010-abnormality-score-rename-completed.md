# ADR-0010 — abnormality_score() Rename Completed

## Статус
Прийнято.

## Рішення
Compatibility-період, описаний в ADR-0006 (risk_score() як тимчасовий alias для abnormality_score()), завершено. Constitution v3.1 фіксує abnormality_score() як canonical назву методу LaboratoryResult — точнішу за risk_score(), оскільки метод оцінює severity відхилення одного показника (Single Result Risk) через дискретні сходинки (0.0/0.25/0.5/0.75/1.0), а не клінічний ризик чи прогноз. Alias risk_score() видалено з коду (entities.py) і з тестів (test_abnormality_score.py, клас TestRiskScoreAlias) після підтвердження через повнотекстовий пошук по репозиторію (Get-ChildItem -Recurse, не PowerShell "**"-glob, який не розкривається рекурсивно), що метод ніде більше не викликається.

## Наслідки
- risk_score() більше не існує в кодовій базі FHOS; будь-яка майбутня згадка цієї назви в новому коді чи документації є помилкою, що вимагає виправлення, а не наміром повернутись до старої назви.
- Precedent неймингу для майбутніх модулів: уникати назв методів, що звучать як клінічний діагноз/прогноз (risk, diagnosis), коли метод фактично виконує технічну класифікацію чи вимірювання (severity, deviation, score).
- Перевірка "чи метод більше ніде не використовується" перед видаленням alias — обов'язковий крок, що має повторюватись при видаленні будь-якого майбутнього compatibility-рішення, зафіксованого за принципом ADR-0006.
