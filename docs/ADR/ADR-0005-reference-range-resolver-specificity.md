# ADR-0005 — ReferenceRangeResolver: специфічність і пріоритети
## Статус
Прийнято.
## Рішення
ReferenceRangeResolver обирає найбільш специфічний ReferenceRange для контексту пацієнта (test_code, unit, sex, age, laboratory_name, method). Специфічність рахується як кількість заповнених обмежуючих полів (sex, age, laboratory_name, method), що співпадають з контекстом; найвищий score перемагає. Якщо кілька кандидатів мають однаковий score — детермінований tie-break за найменшим id, не випадковий вибір. Якщо reference_min/reference_max передані вручну в запиті — Resolver не викликається взагалі (manual override має абсолютний пріоритет).
## Наслідки
- Resolver ніколи не вгадує невідомий контекст: якщо sex/age пацієнта невідомі (None), розглядаються лише діапазони без відповідного обмеження — не найближче наближення.
- Дублікати з однаковою специфічністю — ознака проблеми якості довідникових даних, а не привід для випадкової поведінки; tie-break за id робить це видимим і відтворюваним.
- Ручне введення reference_min/reference_max оператором ніколи не перезаписується автоматично знайденим діапазоном; статус результату явно фіксується як manual/resolved/not_found.
