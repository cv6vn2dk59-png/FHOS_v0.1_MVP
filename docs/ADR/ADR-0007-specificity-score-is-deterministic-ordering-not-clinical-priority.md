# ADR-0007 — Specificity score is deterministic ordering, not clinical priority
## Статус
Прийнято.
## Рішення
ReferenceRange.specificity_score() є алгоритмом детермінованого вибору серед сумісних кандидатів, а не медичною оцінкою клінічної важливості параметра. Рівна вага полів (sex, age, laboratory_name, method — кожне +1) є свідомим технічним компромісом v1, не твердженням, що ці фактори однаково впливають на клінічну точність для будь-якого лабораторного тесту. Тест-специфічні правила ваги (наприклад, вища вага method для одних аналітів, sex для інших) не вводяться зараз — вони вимагають підтвердженої повторюваності на реальних прикладах (TSH, Troponin, Ferritin тощо), як і будь-яке інше архітектурне узагальнення в FHOS.
## Наслідки
- Документація та код повинні уникати формулювань на кшталт "найбільш підходящий" чи "найкращий" Reference Range — коректне формулювання: "найбільш специфічний серед сумісних кандидатів за деструктурованим критерієм".
- Введення тест-специфічної ваги (Strategy Pattern чи інший механізм) вимагає окремого ADR з описом реальних клінічних прикладів, що обґрунтовують потребу — не теоретичного припущення.
- Ця відмінність (deterministic ordering vs clinical priority) має явно поширюватись на будь-яке майбутнє розширення specificity-логіки, включно з Unit Conversion чи Regional Reference Range, якщо вони колись додадуть власні критерії вибору.
