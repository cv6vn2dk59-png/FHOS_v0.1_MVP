# Sprint 7 E06 -- ICD-11 v1: повне дерево (ADR-0015 п.4, закриття)

## Контекст
ADR-0015 п.4 явно залишав повне дерево (17000+ вузлів, увесь WHO
SimpleTabulation MMS) окремим майбутнім рішенням, не автоматичним
розширенням тестової підмножини (розділ 1, S07E02). Користувач явно
обрав цей крок (AskUserQuestion, S07E06) серед відкритих пунктів
проєкту.

## Implementation
- `app/modules/icd11/persistence/who_source_loader.py` --
  `load_icd11_chapter_from_xlsx(chapter_no: str | None = "01")`.
  `chapter_no=None` вимикає фільтр `ChapterNo` повністю. Функція й
  параметр НЕ перейменовані -- мінімальна, зворотно сумісна зміна:
  `chapter_no="01"` і далі працює як у S07E02.
- `scripts/seed_icd11_data.py` -- дефолт `seed()` змінено з
  `chapter_no="01"` на `chapter_no=None`. Повне дерево тепер
  стандартна поведінка без аргументів. Docstring чесно фіксує
  масштаб: ~35000 flush() (по одному на вузол, self-referencing FK
  без ORM relationship(), S07E02) замість ~1000 -- очікується
  довший час виконання, не помилка.
- `tests/modules/icd11/test_who_source_loader.py` -- 4 нових тести
  (`TestFullTreeNoChapterFilter`): включення рядків з кількох глав
  (фікстур уже містив рядок глави 02, навмисно виключений у старих
  тестах), фільтр невідомого ClassKind і далі працює, точний
  підрахунок без фільтра глави, регресія на явному `chapter_no="01"`.

## Архітектурне рішення, зафіксоване явно
Прохід 1 (`BlockId -> URI` мапа) уже сканував увесь файл незалежно
від `chapter_no` (S07E02) -- це не змінюється. Єдина НОВА умова
коректності: self-referencing FK покладається на порядок рядків
файлу (батько раніше дитини) для ВСІХ глав одразу, а не тільки
всередині однієї. Гарантія та сама, що вже підтверджена для розділу 1
(0 unresolved_parents реальним запуском S07E02), але формально мала
бути повторно підтверджена на повному файлі -- див. Verification.

## Verification (підтверджено реальним запуском, 2026-07-09)
Sandbox: 13/13 (9 попередніх importer-тестів + 4 нових). Реальний
venv: `pytest` 245/245 (241 + 4). Реальний повний запуск
`seed_icd11_data.py` на справжньому WHO-файлі
(`SimpleTabulation-ICD-11-MMS-en.xlsx`):
- Прочитано рядків: 36044/36044 (увесь файл).
- Пропущено (невідомий ClassKind): 0.
- Пропущено (невалідний вузол): 0.
- **Нерозв'язаних parent: 0** -- гарантія самопосилального FK
  (батько раніше дитини в порядку рядків файлу), вже підтверджена для
  розділу 1 у S07E02, тепер підтверджена на ВСЬОМУ дереві.
- Ідемпотентність: 1055 вузлів розділу 1 (уже в БД з S07E02)
  розпізнані як дублікати й пропущені; додано 34989 нових.
  1055 + 34989 = 36044 -- точний збіг з повним файлом.

ADR-0015 п.4 закрито остаточно, з реальними числами, не лише
sandbox-прогнозом.
