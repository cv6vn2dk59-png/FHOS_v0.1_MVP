# Common Pitfalls

Реальні інциденти Sprint 3. Мета — не повторювати.

## 1. Код у чаті ≠ код на диску
Найчастіша причина втрачених годин. Claude показує ФАЙЛ у чаті — це не означає, що він застосований. Завжди перевіряй командою (`Get-Content`, `python -c "import ..."`) перед тим, як вважати зміну застосованою.

## 2. Відносні шляхи в PowerShell heredoc — залежать від поточної директорії
`docs\ADR\file.md` створює файл відносно ТОЧКИ ВИКОНАННЯ команди. Виконано з `backend/` замість кореня репозиторію → файл потрапляє в `backend/docs/ADR/`, не в `docs/ADR/`. Використовуй `$root = 'D:\...'` + `"$root\docs\..."` для критичних файлів (документація, конфігурація).

## 3. Циклічний імпорт при реєстрації ORM
`app/persistence/__init__.py` з `from module import Клас` створює цикл, якщо `module` сам стає першою точкою входу процесу. Рішення: окремий `model_registry.py` з `import module` (без вибірки класу).

## 4. SQLAlchemy Enum серіалізує через .name (UPPERCASE), не .value
За замовчуванням `SAEnum(MyEnum)` зберігає ім'я члена enum, не значення. Якщо CHECK constraint чи Domain очікують lowercase — потрібен явний `values_callable=lambda enum_cls: [m.value for m in enum_cls]`.

## 5. PRAGMA foreign_keys — вимкнено за замовчуванням у SQLite
Без `event.listens_for(engine, "connect")` з `PRAGMA foreign_keys=ON`, FK constraints існують у DDL, але не забезпечуються реально. `ON DELETE SET NULL` не спрацює, `IntegrityError` не викинеться.

## 6. IntegrityError вилітає на flush(), не тільки на commit()
`Repository.add()` викликає `session.flush()` для `refresh()` — це матеріалізує INSERT негайно. `try/except` навколо самого лише `commit()` пропустить виняток, що стався раніше. Обгортай весь блок `add()` + `commit()`.

## 7. Get-Content без -Encoding UTF8 — мойбейк кириличних файлів
PowerShell 5.1 читає файли без BOM через системний codepage, не UTF-8. Файл на диску коректний, консоль показує спотворений текст. Виправлення — раз і назавжди в `$PROFILE`, не прапорцем щоразу.

## 8. Ручне редагування файлу-фрагментом замість повного файлу
"ДІЯ: оновити (тільки метод X)" сприйняте як "замінити весь файл цим фрагментом" — губить решту файлу (імпорти, інші методи). За замовчуванням давати повний файл, не фрагмент, коли є найменший сумнів.

## 9. Порожня директорія-артефакт від невдалої спроби
Якщо файл спершу створено не туди (Pitfall 2), а потім успішно перестворено в правильному місці — стара директорія з дублікатом лишається на диску. Git її не бачить (порожні директорії не трекаються), але вона засмічує диск. Прибирай явно `Remove-Item -Recurse -Force`.
